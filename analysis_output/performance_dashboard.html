Repetition Analysis - Data Quality Validation

Individual Repetition Metrics

Shows raw performance data for each individual benchmark run within each configuration.

What it contains:
• repetition: Run number (4, 3, 2, 1 in descending order)
• throughput_mean: Average documents indexed per second during that run
• latency_p50/p90/p99: 50th, 90th, and 99th percentile response times in milliseconds
• error_rate: Fraction of failed requests (0.000 = no errors)
• duration: How long the benchmark run took

Purpose:
• Spot individual run anomalies before they get averaged out
• See run-to-run variation within the same configuration
• Identify trends across repetitions (getting better/worse over time)
• Validate data quality by checking if all runs are reasonably similar

Coefficient of Variation Analysis

In the dashboard:
• throughput_mean_cv: How consistent throughput is across repetitions
• latency_p50_cv, latency_p90_cv, latency_p99_cv: How consistent latency percentiles are
• cpu_avg_cv: How consistent average CPU utilization is across repetitions
• cpu_peak_cv: How consistent peak CPU spikes are across repetitions
• load_avg_1m_cv: How consistent system load averages are across repetitions
• total_queue_cv: How consistent thread pool queue depths are across repetitions
• total_rejected_cv: How consistent thread pool rejections are across repetitions
• max_queue_cv: How consistent peak queue depths are across repetitions

Interpretation:
• CV < 5%: Very consistent (good) - Green
• CV 5-10%: Moderately consistent - Yellow
• CV > 10%: High variability (investigate causes) - Red

What Each Column Tells You:
• Low throughput_mean_cv: Reliable, repeatable performance results
• Low latency_cv values: Predictable response times across runs
• Low cpu_avg_cv: Stable CPU usage patterns, no random spikes or bottlenecks
• Low cpu_peak_cv: Predictable peak CPU loads, consistent workload handling
• Low load_avg_1m_cv: Stable system load, no interference from other processes
• Low queue_cv values: Consistent thread pool behavior, no unexpected queueing or rejections
• High CV values (>10%): Inconsistent behavior - may indicate system interference, thermal throttling, resource contention, or configuration issues
